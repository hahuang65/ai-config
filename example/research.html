Fail-open on Redis outage. If Redis is unreachable, the rate limiter allows all requests rather than blocking the entire API. Multi-worker deployment (4 uvicorn workers) requires shared state in Redis — in-memory counters would allow 4x the intended limit. All increment-and-check operations must be atomic (Redis pipeline) to prevent race conditions under concurrent load.
