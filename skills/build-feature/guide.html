/build-feature Pipeline Guide

Complete checkpoint reference for testing the 3-phase workflow

1 Research → 2 Plan → 3 Implement

02 Master Pipeline Checklist

Phase # Task / Checkpoint Agent / Command Req? Observable Output
PHASE 1 — RESEARCH
Research 1.1 Create feature directory docs/claude/<timestamp>-<slug>/ — MUST Directory created on disk
Research 1.2 Identify scope from $ARGUMENTS — MUST Scope statement in research.md overview
Research 1.3 Deep-read code (every file in target area, trace data flows, examine tests, check configs) — MUST Extensive Read/Grep tool calls visible in transcript
Research 1.4 Write research.md with 8 required sections: Overview, Architecture, Key Files, Data Flow, Patterns, Dependencies, Edge Cases, Current State — MUST File written with all 8 sections populated
Research 1.5 Invoke /generate-web-diagram → architecture.html /generate-web-diagram COND HTML file created, opened in browser (only if VE available)
Research 1.6 STOP — wait for user review ("move to planning") — MUST Message: "Phase 1 complete. Please review..."
PHASE 2 — PLAN
Plan 2.1 Read context — check docs/claude/ for research.md, read relevant source files — MUST Read calls to research.md and source files
Plan 2.2 Architecture review architect MUST Agent tool call to architect agent
Plan 2.3 Detect domain context — invoke frontend-design, frontend-patterns, api-design skills if applicable domain skills COND Skill invocations for matching domains (if installed)
Plan 2.4 Write plan.md with required sections: Goal, Research Ref, Approach, Detailed Changes, New Files, Dependencies, Considerations, Migration, Testing Strategy — MUST File written with all sections; code snippets (not pseudocode)
Plan 2.5 STOP — wait for user annotation (// comments) — MUST Message: "The plan is ready for your review..."
Plan 2.6 Address ALL // annotations, update plan, remove processed annotations — MUST Plan.md updated, no remaining // lines
Plan 2.7 Repeat annotation cycle until user approves (1-6 iterations typical) — MUST Cycle continues until explicit approval
Plan 2.8 Generate Todo List (test tasks mandatory, grouped by phase) — MUST Todo List section appended to plan.md with - [ ] items
Plan 2.9 Invoke /generate-visual-plan → visual-plan.html /generate-visual-plan COND HTML file created, opened in browser (only if VE available)
Plan 2.10 STOP — wait for user to say "implement" — MUST Message: "Phase 2 complete. Say implement..."
PHASE 3 — IMPLEMENT
Impl 3.1 Read the plan document thoroughly — MUST Read call to plan.md
Impl 3.2 Implement ALL tasks via tdd-guide agent (red-green-refactor cycle) tdd-guide MUST Agent tool calls; test-first implementation visible
Impl 3.3 Track progress — mark tasks - [x] in plan.md after each completion — MUST Plan.md checkboxes updated incrementally
Impl 3.4 Maintain code quality — follow existing patterns, no unnecessary comments — MUST Code matches codebase conventions
Impl 3.5 Verification loop: type check + lint + test + build (all 4 must pass) — MUST All 4 commands run and pass; loop repeats on failure
Impl 3.6 Security review on all changed files security-reviewer MUST Agent call; CRITICAL issues fixed immediately
Impl 3.7 Database review (if feature touches DB code) database-reviewer COND Agent call; CRITICAL/HIGH issues fixed
Impl 3.8 Invoke /simplify /simplify MUST Simplify skill runs; re-run tests after
Impl 3.9 Refactor cleanup on changed files refactor-cleaner MUST Agent call; SAFE items removed; re-run tests
Impl 3.10 Code review on all changed files (>80% confidence threshold) code-reviewer MUST Agent call; CRITICAL/HIGH fixed; re-run tests
Impl 3.11 Documentation update (if feature warrants it) doc-updater COND Agent call; READMEs/CHANGELOG updated
Impl 3.12 Invoke /fact-check on plan.md /fact-check MUST Fact-check skill runs on plan file
Impl 3.13 Refresh visual-plan.html if plan changed (deviations, corrections, task changes) /generate-visual-plan COND Regenerated HTML if plan diverged
Impl 3.14 Summary to user — NEVER commit — MUST Message: "Implementation complete." No git commit.
POST-IMPLEMENTATION — BUILD-FEATURE ORCHESTRATOR
Post P.1 Invoke /diff-review → diff-review.html /diff-review COND HTML diff review created, opened in browser (only if VE available)
Post P.2 Invoke /fact-check on diff-review.html /fact-check COND Fact-check runs on generated HTML (only if VE available)

03 Agent Registry

tdd-guide
Test-Driven Development specialist enforcing write-tests-first methodology. Guides red-green-refactor cycle, ensures 80%+ coverage. Reads testing.md and coding-style.md.
sonnet Phase 3 — step 3.2 implement skill MUST
architect
Software architecture specialist for system design, scalability, and technical decision-making. Evaluates trade-offs, proposes ADRs. Reads coding-style.md, performance.md, security.md.
opus Phase 2 — step 2.2 plan skill MUST
security-reviewer
Security vulnerability detection and remediation. Covers OWASP Top 10, secrets detection, input validation, auth/authz. Severity levels: CRITICAL, HIGH, MEDIUM. Reads security.md.
sonnet Phase 3 — step 3.6 implement skill MUST
database-reviewer
Database specialist for query optimization, schema design, RLS, connection management. Runs EXPLAIN ANALYZE, checks indexes and N+1 patterns. Reads security.md, performance.md.
sonnet Phase 3 — step 3.7 implement skill COND: DB code
refactor-cleaner
Code cleanup specialist for dead code detection, unused dependency removal. Categorizes findings: SAFE (remove), CAREFUL (verify), RISKY (confirm). Reads coding-style.md.
sonnet Phase 3 — step 3.9 implement skill MUST
code-reviewer
Expert code review with confidence-based filtering (>80% threshold). Reviews CRITICAL security, HIGH quality, MEDIUM performance, LOW practices. Reads all 4 rule files.
sonnet Phase 3 — step 3.10 implement skill MUST
doc-updater
Documentation specialist ensuring docs match code reality. Updates READMEs, CHANGELOGs, architecture docs. Verifies paths and examples. Reads git-workflow.md.
sonnet Phase 3 — step 3.11 implement skill COND: significant changes
