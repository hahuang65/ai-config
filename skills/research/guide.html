Deliverable

Required Document Sections

Every research.md must include all eight sections. Missing sections are a failure.

Section 1
Overview
What the system or module does at a high level. What problem does it solve? Who are the consumers?
Concise Jargon-free
Section 2
Architecture
How components are organized and relate to each other. Module boundaries, layering, communication patterns.
Diagram-ready Boundaries clear
Section 3
Key Files
Important files with their roles. Absolute paths and brief descriptions of each file's responsibility.
Absolute paths Line refs
Section 4
Data Flow
How data moves through the system end-to-end. Inputs, transformations, storage, serialization boundaries, outputs.
End-to-end Traced
Section 5
Patterns & Conventions
Design patterns, naming conventions, error handling style, configuration approach, testing patterns used.
Specific examples Descriptive
Section 6
Dependencies
Internal module imports and external packages. Runtime services, version constraints, and why they matter.
Internal External
Section 7
Edge Cases & Gotchas
Non-obvious behavior, implicit assumptions, potential pitfalls. Things that would bite someone unfamiliar with the code.
Explicit callouts Honest
Section 8
Current State
Known issues, tech debt, TODOs, areas of concern. An honest assessment of what works and what needs attention.
Tech debt flagged Actionable

Standards

Quality Checklist

  • Exhaustive — too much detail is better than too little. Surface-level skimming is not acceptable. Every file in scope should be read and referenced.
  • Specific file paths and line references — absolute paths, not vague descriptions like "the auth module." Include line numbers for key findings.
  • Calls out unclear or buggy things — flag smells, tech debt, confusing patterns, undocumented behavior, implicit assumptions explicitly.
  • No code changes — research is read-only. Zero modifications to the codebase during this phase.
  • No solutions proposed — research observes and reports. Planning and implementation come in later phases.
