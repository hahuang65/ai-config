detailed layers below
Layer 1 — Skills

User-invocable workflows defined as skills/<name>/SKILL.md. Registered as slash commands. Orchestrate agents and produce persistent markdown artifacts.

/build-feature
Orchestrator. Chains research → plan → implement. Creates docs/claude/ feature directories. disable-model-invocation: true
/research
Deep-reads codebase. Produces research.md + architecture diagram. Model: opus
/plan
Creates implementation plan with annotation cycles. Loads domain skills. Invokes architect agent. Model: opus
/implement
Executes approved plan. Batches tdd-guide 3-5 tasks. Runs 13-step quality pipeline. Model: sonnet
/visual-explainer
Generates self-contained HTML diagrams. Optional companion at each phase transition. Has templates + CSS references.
/refactor NEW
User-directed code refactoring. Analyze → plan → transform → verify. Invokes refactorer agent. Model: opus
skills invoke agents via Agent tool
Layer 2 — Agents

Specialized subagents defined as agents/<name>.md. Invoked via subagent_type matching agent name. Each declares explicit tool permissions.

architect
System design, ADRs, trade-off analysis. Read-only tools. Model: opus. Invoked from /plan
tdd-guide
Red-green-refactor TDD cycle. 80%+ coverage. Batched 3-5 tasks from /implement
code-reviewer
OWASP Top 10, quality, performance review. Confidence >80%. Auto-fixes CRITICAL issues
refactor-cleaner
Dead code detection + safe removal. SAFE/CAREFUL/RISKY risk categories. Post-implementation cleanup
database-reviewer
Query optimization, N+1 detection, EXPLAIN ANALYZE. Conditional: only if DB code touched
doc-updater
README, CHANGELOG, architecture docs sync. Verifies paths and examples are current
refactorer NEW
Executes structural transformations: extract, inline, move, rename, restructure. Tests between each step. Model: sonnet
agents read and enforce rules
Layer 3 — Rules

Always-on constraints at rules/*.md

  • coding-style.md <400 LOC, <50 LOC funcs, immutable, no dead code
  • testing.md TDD default, 80%+ coverage, behavioral tests
  • security.md No secrets, parameterized queries, input validation
  • performance.md Profile first, model routing, caching
  • git-workflow.md TYPE: Subject, feature branches, docs/claude/
  • development-workflow.md Plan → TDD → Review → Commit
Reference Skills

Conditionally loaded by /plan when domain context detected

  • api-design REST patterns, pagination, error responses, versioning
  • frontend-patterns Components, state management, a11y, performance

infrastructure
Infrastructure
install.sh
Symlinks skills, agents, commands, rules to ~/.claude/. Skips commands with matching skills.
test-pipeline.sh
Validates frontmatter, cross-references, tool names, agent dependencies, stale stubs. Must pass for new components.
settings.json
Model selection (opus default), permissions (allow/deny/ask), MCP plugins
hooks.json
Session stop hooks (cost tracker). Extensible for pre-commit validation.
Key insight: The new /refactor skill fills a gap — existing agents find problems (code-reviewer) or clean up after implementation (refactor-cleaner, /simplify), but none perform intentional, user-directed structural transformations with step-by-step behavior verification. The refactorer agent executes specific transformations (extract, inline, move, restructure) while the skill orchestrates scope confirmation, incremental testing, and the verification loop.
