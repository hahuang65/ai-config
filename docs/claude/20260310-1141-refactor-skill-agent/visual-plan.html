Safety Rules & Edge Cases
ScenarioExpected Behavior
Tests fail before refactoringSTOP immediately. Report broken baseline. Do not refactor broken code.
Transform causes test failureRevert ALL changes from that transform. Report which and why. Continue only if remaining transforms are independent.
Refactoring would change behaviorSTOP and report the behavioral change. Do not proceed without user approval.
Existing test needs modificationSTOP and present required test changes to user for approval. Never silently modify tests.
Scope exceeds 15 filesSkill warns and suggests breaking into phases. User must explicitly confirm to proceed.
Vague refactoring goalAsk for clarification rather than guessing. Don't interpret "clean up" as license to rewrite.
Public API signature changeOnly if user explicitly requested. Classify as RISKY. Verify all external callers.
Rename with dynamic referencesGrep for string interpolation, reflection, eval patterns. Flag if uncertain.

Implementation Phases
Phase 1: Agent
Write agents/refactorer.md: frontmatter, opening, rules, test preservation rule, categories table, risk table, workflow, safety rules, revert protocol, review checklist
Phase 2: Skill
Create skills/refactor/ directory. Write SKILL.md: frontmatter, parse goal, read code, detect tooling, present plan, STOP point, execute agent, verify, report, scope control, feedback loop
Phase 3: Validate
Run test-pipeline.sh — all checks pass. Run install.sh — symlinks created. Verify /refactor appears as slash command.
Critical: Test preservation. The refactorer agent MUST NOT modify or remove any existing tests. If a refactoring breaks an import in a test file, the agent stops and presents the change for user approval. This prevents silent behavioral changes hidden behind "updated tests."
Scope guard. No persistent artifacts produced — the transformed code is the deliverable. No docs/claude/ directory created. No commits. The git diff tells the story.
Composability. Standalone by design. Does not integrate with /implement or /build-feature. The refactor-cleaner agent handles post-implementation cleanup (different concern). This skill handles intentional structural improvements.
