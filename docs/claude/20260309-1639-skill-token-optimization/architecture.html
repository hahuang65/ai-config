Redundancy Hotspots

#1: VE SKILL.md Reloads

~166K

The 33K visual-explainer skill reloads 5 extra times after its first load. Each reload is pure waste — the instructions don't change between invocations.

33,179 chars x 5 redundant loads = 165,895 chars

#2: Reference File Re-reads

~315K

css-patterns.md (42K) and libraries.md (19K) are re-read 4-5 times each. Templates (13-17K) are re-read 3-4 times. Same content, no changes between reads.

~78K per read x 4 redundant reads = ~315K chars

#3: fact-check Loads Full VE

~66K

fact-check only needs css-patterns.md for HTML styling. Loading the full 33K VE skill is unnecessary — it adds diagram-making instructions irrelevant to fact-checking.

33K per invocation x 2 invocations = ~66K wasted

#4: Cross-Command Duplication

~5-7K

Overflow prevention, Mermaid zoom, verification checkpoint, output location, and surf illustrations repeat across 5+ command files verbatim.

~300-400 chars x 4-6 appearances each = ~5-7K total
Key insight: The VE reload problem accounts for ~75% of all token waste. Eliminating just VE SKILL.md reloads and reference re-reads would reduce total /build-feature token cost from ~681K to ~200K — a 70% reduction. Agents run in their own context, so their sizes don't affect the main conversation budget.
