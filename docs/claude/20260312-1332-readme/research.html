Phase Details

Phase 1: Research
  • Create feature dir: docs/claude/<timestamp>-<slug>/
  • Deep-read target codebase (every file)
  • Write research.md
  • Generate research.html via visual-explainer
  • STOP — wait for user review
  • Annotation cycle: address // notes, regen HTML
Phase 2: Plan
  • Read research + source files
  • Invoke architect agent opus if architectural
  • Auto-load frontend-patterns / api-design
  • Write plan.md with Proposed Todo List
  • Generate plan.html before first review
  • STOP — annotation cycles (1-6 rounds)
  • On approval: finalize todo list, regen HTML
Phase 3: Implement
  • TDD via tdd-guide agent sonnet
  • Verify loop: typecheck, lint, test, build
  • database-reviewer sonnet if DB touched
  • refactor-cleaner sonnet dead code
  • code-reviewer sonnet OWASP review
  • doc-updater sonnet if API changed
  • Refresh plan.html + generate diff-review.html
  • NEVER commits — user decides

Dual-Tool Configuration

Claude Code
Full feature set. Skills at ~/.claude/skills/, commands at ~/.claude/commands/, agents at ~/.claude/agents/, rules at ~/.claude/rules/. settings.json is the source of truth for permissions.
OpenCode
Shared skills (same ~/.claude/skills/), commands at ~/.config/opencode/commands/. No agents, rules, or hooks support. Permissions auto-generated from Claude via sync-permissions.py.
Permission Sync — claude/settings.json is edited directly. On every commit, the pre-commit hook runs sync-permissions.py, which converts Claude's Tool(pattern) format to OpenCode's "pattern": "level" format, merges with config/opencode-only.json, and writes opencode/opencode.jsonc. Never edit the OpenCode config directly.
