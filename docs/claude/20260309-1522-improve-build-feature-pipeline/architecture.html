6 Identified Gaps

What ECC has that your pipeline doesn't

1

No Automated Hooks

Quality checks (formatting, type checking, security scans) only happen when the implement skill explicitly says to. No automatic triggers after edits.
ECC: PostToolUse hooks auto-format, typecheck, and run quality gates after every file edit
2

No Domain-Knowledge Skills

When planning an API or frontend feature, the AI has no embedded reference for patterns, status codes, pagination, component composition, etc.
ECC: api-design, frontend-patterns, backend-patterns, e2e-testing skills provide reusable domain knowledge
3

No Model Routing

Every phase runs on whatever model the user selected. No cost optimization for lighter tasks or quality optimization for complex reasoning.
ECC: Agent definitions specify model tiers — Opus for planning, Sonnet for reviews, Haiku for workers
4

No Always-On Rules

Coding standards depend entirely on the project's CLAUDE.md. If it's missing or sparse, no baseline quality standards are enforced.
ECC: 9 common-* rules with alwaysApply: true set immutability, testing, security, and style standards globally
5

No Security Review Step

The implement phase does simplify and fact-check, but never runs a security review. OWASP Top 10, hardcoded secrets, and injection patterns go unchecked.
ECC: security-reviewer agent runs automatically before commits, checking OWASP Top 10 and secret detection
6

No Cost Tracking or Learning

No visibility into how many tokens each pipeline phase consumes. No session-end learning to extract reusable patterns from completed work.
ECC: cost-tracker.js tracks tokens per session; evaluate-session.js extracts patterns; continuous-learning hooks observe tool use
