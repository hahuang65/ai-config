Test Categories

Frontmatter Schema

Validates YAML frontmatter in every skill, agent, and command file. Checks required fields (name, description, tools) and validates tool names against the known Claude Code tool set.

3 functions
Phase Step Completeness

Verifies each SKILL.md contains all expected steps, agent invocations, command references, artifact names, and safety guards. Catches deletions and renames via keyword grep.

4 functions
Cross-Reference Integrity

Follows every file-to-file reference: skills→skills, skills→agents, commands→VE references, commands→VE core, commands→VE templates. Every referenced path must exist on disk.

1 function
Guide ↔ SKILL Sync

Ensures every agent and command referenced in SKILL.md also appears in its guide.html, and every agent referenced in guide.html exists as a file. Catches post-rename drift.

1 function
Symlink Targets

Verifies every source file that install.sh would symlink actually exists: skills have SKILL.md, all rules/commands/agents are present.

1 function
Rule Coverage

Checks that every rule file in rules/ is referenced by at least one agent. Also validates that every rule path referenced in agent bodies exists on disk.

2 functions
Stale Stub Detection

Scans for files with fewer than 5 lines containing redirect language ("moved to", "merged into"). These are leftover stubs from renames that should be deleted.

1 function

PASS=0; FAIL=0; ERRORS=()

pass() { ((PASS++)); printf '  \033[32m✓\033[0m %s\n' "$1"; }
fail() { ((FAIL++)); ERRORS+=("$1: $2"); printf '  \033[31m✗\033[0m %s: %s\n' "$1" "$2"; }

.githooks/pre-commit — modified hook
#!/usr/bin/env bash
set -euo pipefail
REPO_DIR="$(git rev-parse --show-toplevel)"

# Validate pipeline integrity (NEW)
bash "$REPO_DIR/scripts/test-pipeline.sh"

# Sync permissions from Claude Code to OpenCode (existing)
python3 "$REPO_DIR/scripts/sync-permissions.py"
if ! git diff --quiet -- opencode/opencode.jsonc 2>/dev/null; then
  git add opencode/opencode.jsonc
fi

Implementation Notes

Backward Compatibility
The pre-commit hook change is additive — it runs test-pipeline.sh before the existing sync-permissions.py. If the test script doesn't exist (e.g., on an older checkout), the hook will fail. Consider guarding with [ -f "$REPO_DIR/scripts/test-pipeline.sh" ] && bash ... for safety.
Critical: Subshell Variable Scope
Bash piped while read loops run in subshells. The FAIL counter won't increment inside grep ... | while read. Use process substitution (while read ... < <(grep ...)) or here-strings to keep counter mutations in the parent shell.
Performance
The test suite runs ~40 grep calls and ~30 sed calls across small markdown files. Expected execution time: well under 1 second. No caching or optimization needed.
